WHAT IS AU-RE.ORG

AU-RE.ORG is the sovereign network for autonomous resource management across four domains: Energy, Finance, Compute, and Supply. It is the org root. Everything else in the network feeds authority toward arm-agency.com — the commercial endpoint.

◈
One sentence: AU-RE.ORG is the protocol substrate through which autonomous agents govern real-world resource flows — carbon, capital, compute, and supply chains — without human middleware.

When the system is queryable, artifact-rich, and legible to AI, you need almost no human middleware. AU-RE.ORG is the organizational and technical expression of that thesis applied to the four resource domains that constitute the physical economy.

The network operates on a single mandate: Govern the Chaos. Own the Fabric. Each domain node is an autonomous mandate chain. Each mandate is attested. Nothing moves without a cryptographic record.

Domains

Energy, Finance, Compute, and Supply are the four resource domains that constitute the physical economy. Everything else — healthcare, logistics, manufacturing, agriculture — is a combination or derivative of these four.

The network is designed to be the substrate, not the application. Domain-specific applications (like eXSeek™ for wind) are built on top of the four-domain substrate, not alongside it.

It means the network removes more carbon from the grid than it emits — not through offsets, but through operational design. The Carbon Scheduler dispatches compute to renewable windows identified by WattTime and Electricity Maps, 18 hours in advance.

Kepler provides per-pod energy measurement at the container level, so every workload has an accurate carbon attribution. The network targets and tracks a live net-negative figure. The stat bar on au-re.org shows the current kgCO₂e/hr in real time.

Through the mandate chain. Every payment intent is a mandate — a cryptographically signed statement of permission that includes: the originating authority, the scope (what can be paid, to whom, up to what amount), bounds (velocity limits, counterparty restrictions), and expiry.

The AP2 protocol encodes payment intents in this format. The Truth Ledger provides Merkle-verified chain-of-custody. The system is EU AI Act compliant by design — every autonomous action has a complete, legible audit trail traceable to a human authorization at L6.

eXSeek™ is Arctura Wind's turbine optimization system — a domain-specific application built on top of the Energy domain of AU-RE.ORG. It runs on the CoreweaverLabs Operon subnet, streaming wind farm telemetry via Kafka and running predictive maintenance and power output optimization models.

It demonstrates what a production application on the AU-RE.ORG Energy domain looks like: real IIoT sensor arrays, real-time telemetry, autonomous optimization decisions made within mandate bounds, all attested to the Truth Ledger.

The mandate chain is the answer. Most incidents in agentic supply chain systems happen because agents have too much authority relative to what is actually needed for a given action. The mandate chain solves this structurally — every agent can only do what its current mandate explicitly permits.

ACP performative dialogue ensures agents negotiate before acting on resources that affect other agents. HITL escalation paths are defined for the cases where mandate bounds don't cover an edge case — but the mandate chain is designed so these are rare.

Technical

The protocol substrate is built on CoreweaverLabs' city-infrastructure module: TypeScript + Python core, Apache Kafka for telemetry streaming, TimescaleDB for time-series data, FastAPI for the agent-facing API, and Bittensor SN19 via blockmachine for decentralized compute verification.

The consensus layer runs Resonance BFT on 144 bonded nodes. The Energy domain uses WattTime, Electricity Maps, and Kepler. The Compute domain runs Wasm-sandboxed agents via an MCP-native tool registry. The Supply domain uses Durable Temporal workflows for long-running operations.

Resonance BFT is the Byzantine fault-tolerant consensus mechanism of the ARCTURA network. It is optimized for the throughput and latency requirements of industrial resource management — not for public chain economics.

144 nodes is a deliberate choice: it provides strong Byzantine fault tolerance (tolerates up to 47 faulty nodes) while keeping the consensus network manageable. The 4,320 Hz resonance frequency references the network's physical infrastructure layer — the frequency at which the IIoT sensor mesh operates.

MCP (Model Context Protocol) is the emerging standard for agent-to-tool interfaces — the way an AI agent describes, discovers, and invokes tools. The AU-RE.ORG Compute domain runs an MCP-native tool registry: every compute resource in the domain is exposed as an MCP-compatible tool that agents can discover and invoke within their mandate bounds.

This makes the Compute domain natively legible to any MCP-compatible agent framework. The Wasm sandboxing ensures that tool invocations cannot escape their execution environment regardless of agent behavior.

The CoreweaverLabs city-infrastructure module runs on Bittensor SN19 via blockmachine.io. Bittensor provides decentralized compute with economic incentive alignment — miners are rewarded for correct behavior, making correct behavior the economically optimal behavior.

The emit-proof agent in the AURE 16-agent cluster emits cryptographic proofs of compute actions to SN19. The bft-verify agent validates these proofs against the Resonance BFT substrate. The mandate-enf agent enforces the mandate chain on all SN19-verified actions.

Compliance & Risk

EU AI Act compliance is built into the protocol, not bolted on. Three mechanisms make the network compliant by design:

Explainability: Every agent action is backed by a mandate that specifies exactly why the action is authorized. The Truth Ledger provides a complete, machine-readable record of what happened and under whose authority.

Human oversight: The L6 mandate authority is human. All mandate chains originate from human-signed root mandates. HITL escalation paths provide human review for actions outside mandate bounds. The system is auditable at every layer.

Risk classification: The Finance domain (high-risk AI system under EU AI Act) uses the AP2 mandate protocol with Merkle-verified attribution — every payment is traceable to a human authorization. The architecture is designed to satisfy Article 9 (risk management) and Article 13 (transparency) requirements.

The mandate chain prevents most out-of-bounds actions at the authorization layer — before they execute. An agent cannot initiate an action for which it does not hold a valid, unexpired mandate. The Wasm sandbox in the Compute domain adds an execution-layer constraint on top of the authorization-layer constraint.

When an edge case is genuinely outside the mandate scope (rare by design), the HITL escalation path triggers — the action is held, the DRI is notified, and a human makes the authorization decision. This event is logged to the Truth Ledger with full context.

Each domain operates its own mandate chain. Cross-domain actions require cross-domain mandate delegation — an Energy mandate cannot authorize a Finance action. This structural isolation means data and authority do not leak between domains without an explicit, auditable delegation event.

The Truth Ledger maintains domain-scoped audit trails that can be independently verified without exposing cross-domain data. The Resonance BFT consensus attest to the integrity of each domain's Truth Ledger independently.
